Business Logic Endpoint URL HTTP Method Query Parameters Request Body Data Format Response Body Data Format
Return a list of ALL movies to the user /movies GET None None A JSON Object holding data about all movies.
Return data about a single movie by title to the user /movies/[Title] GET [Title] as the title/name of the movie None A JSON Object holding data about a single movie, example:
        {
          "Title": "Movie",
          "Description": "Description",
          "Genre": {
            "Name": "Genre",
            "Description": "Description",
          },
          "Director": {
            "Name": "Director",
            "Bio": "Bio",
            "Birthyear": "1990"
          },
          "ImageURL": "pic.jpeg",
          "Featured": true
        }
Return data about a genre by name/title /movies/genre/[Genre] GET [Genre] being name of genre, like Thriller None A JSON Object holding data about a specific genre, example:
          {
            "Name": "Genre",
            "Description": "Description"
          }
Return data about a director by name /movies/directors/[Name] GET [Name] being the name of the director None A JSON Object holding data about a specific director, example:
          {
            "Name": "Director",
            "Bio": "Description",
            "Birthyear": "1990"
          }
Allow new users to register /users POST None A JSON Object structured like:
          {
            "Username": "Name",
            "Password": "Password",
            "Email": "e@mail.com"
          }
 A JSON Object holding this new user's data, including generating a unique id, example:
          {
            "Username": "Name",
            "Password": "Password",
            "Email": "e@mail.com",
            "FavoriteMovies": [],
            "_id": "xxx"
          }
Allow users to update their user info /users/[username] PUT [username] being the username of an existing user A JSON Object with the updated information, like:
          {
            "Username": "Name",
            "Password": "Password",
            "Email": "e@mail.com"
          }
A JSON Object holding the user's updated data, example:
          {
            "Username": "Name",
            "Password": "Password",
            "Email": "e@mail.com",
            "FavoriteMovies": [],
            "_id": "xxx"
          }
Allow users to add a movie to their list of favorites /users/[username]/[id] POST [username] of the user, followed by the movie [id] which can be found in its JSON object None A JSON Object of the user, with FavoriteMovies updated.
Allow users to remove a movie from their list of favorites /users/[username]/[id] DELETE [username] of the user, followed by the movie [id] which can be found in its JSON object None A JSON Object of the user, with FavoriteMovies updated.
Allow existing users to deregister /users/[username] DELETE [username] being the existing user's username None A text message confirming that the user has been deleted.
